1. 概述

Seisei Helpdesk 是一套基于 Odoo 18 的智能客服工单系统，提供：

  • 多团队工单管理 — 支持看板、列表、表单多视图
  • 自动分配算法 — 手动、轮询、负载均衡、技能/标签匹配
  • 可视化仪表盘 — 实时统计我的工单、绩效、各团队情况
  • 多渠道接入 — LINE、微信、WhatsApp、邮件、网页表单
  • SLA 管理（可选模块）— 自定义响应/解决时限，自动追踪

2. 初始配置

2.1 模块一览

模块功能必须
seisei_helpdesk核心工单管理、仪表盘是
seisei_helpdesk_slaSLA 策略与追踪否
seisei_helpdesk_portal客户自助门户否
seisei_messaging多渠道消息基础层接入 LINE 时需要
seisei_messaging_lineLINE Messaging API 适配器接入 LINE 时需要

2.2 创建团队

  1. 进入 Helpdesk → Configuration → Teams
  2. 点击 New
  3. 填写：
    • 团队名称（例：客服一组）
    • 负责人 — 团队管理者
    • 分配方式：
      选项说明
      手动工单不自动分配，需人工指定
      轮询按顺序依次分配给成员 A→B→C→A...
      负载均衡分配给当前打开工单最少的成员
      技能/标签根据工单标签匹配有对应技能的成员
    • 成员 — 添加团队成员
  4. 切换到 Stages 页签 — 配置工作流阶段
  5. 点击 Save

2.3 配置阶段

进入 Configuration → Stages：

字段说明
名称阶段名称（如：新建、处理中、已解决）
顺序看板中的排列顺序
折叠是否在看板中默认折叠
关闭阶段勾选后，工单到达此阶段视为"已关闭"
邮件模板工单进入此阶段时自动发送的邮件通知

2.4 创建标签

进入 Configuration → Tags，创建分类标签（如：技术问题、计费问题、退货、咨询）。标签用于工单分类和技能分配。

3. 仪表盘

仪表盘是 Helpdesk 的首页，点击 Helpdesk → Dashboard 即可进入。

3.1 我的工单（My Tickets）

显示三张 KPI 卡片：

卡片颜色含义
所有工单蓝色分配给我的所有未关闭工单
高优先级黄色优先级为"高"的工单
紧急红色优先级为"紧急"的工单

每张卡片还显示：

  • 平均打开时长 — 工单从创建到现在的平均小时数
  • SLA 超期 — SLA 未达标的工单数量（需安装 SLA 模块后显示）
点击任意卡片可跳转到对应的工单筛选列表。

3.2 我的绩效（My Performance）

指标说明
今日关闭今天关闭的工单数量
每日目标进度条显示完成比例（默认目标：5件/天）
SLA 达标率今日已关闭工单中 SLA 达标的百分比
最近7天关闭过去7天关闭的工单总数
日均7天平均每天关闭数量
7天SLA达标率过去7天的 SLA 达成百分比

3.3 客户关怀（Customer Care）

按团队显示卡片，每张卡片包含四个关键数字：

指标说明点击后
打开该团队当前未关闭工单数跳转到该团队的打开工单列表
未分配未指派负责人的工单数跳转到未分配工单列表
紧急紧急优先级工单数跳转到紧急工单列表
SLA超期SLA 未达标工单数需安装 SLA 模块后显示

4. 工单管理

4.1 创建工单

方法一：手动创建

  1. 进入 Helpdesk → Tickets → All Tickets
  2. 点击 New
  3. 填写：主题、团队、优先级、客户信息、描述
  4. 保存后工单自动编号（如 HD-0001）

方法二：LINE/消息渠道自动创建

  • 需安装桥接模块 seisei_helpdesk_messaging
  • 客户通过 LINE 发送消息时可自动生成工单

方法三：邮件创建

  • 配置团队的邮件别名
  • 客户发送邮件到该地址自动生成工单

4.2 工单视图

  • 看板视图 — 按阶段分列显示，拖拽即可移动阶段
  • 列表视图 — 批量操作，支持排序、分组、筛选
  • 表单视图 — 工单详情，包含聊天记录、活动计划

4.3 常用操作

操作说明
分配给我一键将工单分配给当前用户
修改阶段看板中拖拽 或 表单中选择下拉菜单
修改优先级低 / 中 / 高 / 紧急
发送邮件通过消息区域与客户沟通
记录活动安排待办事项和跟进

4.4 自动分配

当工单创建时，系统根据团队设置自动分配：

  • 轮询：按固定顺序循环分配
  • 负载均衡：分配给当前工单最少的成员
  • 技能/标签：根据工单标签匹配成员，未匹配则回退到负载均衡

5. 团队管理

进入 Configuration → Teams，选择团队编辑：

设置项说明
成员添加/删除团队成员
分配方式选择自动分配算法
工作时间关联工作日历（用于 SLA 计算）
使用 SLA启用后可配置 SLA 策略
客户评价启用后工单关闭时发送满意度调查
自动关闭(天)无活动 N 天后自动关闭工单（0=禁用）
标签分配规则技能模式下配置标签 → 成员映射

6. LINE 消息集成

6.1 前提条件

  1. 拥有 LINE Developers 账号
  2. 创建一个 Messaging API Channel
  3. 获取 Channel Access Token 和 Channel Secret

6.2 配置步骤

Step 1: 创建消息渠道

  1. 在 Odoo 中搜索进入 Messaging 菜单（安装 seisei_messaging 后可见）
  2. 进入 Channels，点击 New
  3. 填写：
    • 名称：LINE 客服
    • 渠道类型：选择 LINE
    • 配置 (JSON)：
{
  "channel_access_token": "你的 LINE Channel Access Token",
  "channel_secret": "你的 LINE Channel Secret"
}

Step 2: 生成 Webhook Token

  1. 在渠道表单中，点击 Generate Webhook Token 按钮
  2. 系统自动生成 Webhook URL
  3. 复制 URL（格式：https://your-domain/seisei/messaging/line/{id}/webhook）

Step 3: 配置 LINE Developers 后台

  1. 登录 LINE Developers Console
  2. 选择你的 Messaging API Channel
  3. 设置：
    • Webhook URL：粘贴步骤2的 URL
    • Use webhook：开启
    • Auto-reply messages：关闭（由 Odoo 接管回复）

Step 4: 验证

  1. 用 LINE 向你的 Bot 发送一条消息
  2. 在 Odoo Messaging → Conversations 中应出现新的会话记录
注意：Webhook URL 必须是公网可访问的 HTTPS 地址。LINE 服务器需要能连接到你的 Odoo 实例。

6.3 功能说明

功能说明
接收消息LINE 用户发消息 → Webhook → Odoo 自动创建会话和消息记录
发送消息在会话中点击"发送消息" → 通过 LINE Push API 推送给用户
用户识别自动获取 LINE 用户名和头像
消息类型支持文字、图片、视频、音频、文件、位置、贴图
AI 自动回复配置 Dify App ID 后，可启用 AI 智能回复

7. SLA 管理

需安装 seisei_helpdesk_sla 模块。

7.1 创建 SLA 策略

  1. 进入团队设置，启用 使用 SLA
  2. 配置 SLA 策略：
    • 名称：如"紧急工单24小时响应"
    • 优先级：指定适用的优先级等级
    • 目标阶段：工单需到达的阶段
    • 时限：小时数（基于工作日历计算）

7.2 SLA 追踪

  • 系统自动计算每张工单的 SLA 状态：进行中 / 已达标 / 超期
  • 仪表盘自动显示 SLA 达标率和超期数量
  • 工单表单中可查看每条 SLA 策略的截止时间和状态

8. 常见问题 (FAQ)

Q: 仪表盘显示"No teams configured"怎么办？

A: 需先创建至少一个团队。进入 Configuration → Teams → New。

Q: LINE 消息收不到怎么排查？

A:

  1. 检查 LINE Developers 后台的 Webhook URL 是否正确
  2. 确认 Webhook 已开启 (Use webhook = ON)
  3. 检查 Channel Access Token 和 Secret 是否正确粘贴到 JSON 配置中
  4. 查看 Odoo 服务器日志中的 webhook 相关错误
  5. 确认 Odoo 服务器域名可被外网访问（HTTPS）

Q: 工单自动分配不生效？

A:

  1. 确认团队分配方式不是"手动"
  2. 确认团队有至少一名成员
  3. 技能/标签模式下，确认配置了标签 → 成员映射

Q: SLA 数据在仪表盘中不显示？

A: 需安装 seisei_helpdesk_sla 模块。安装后，SLA 相关指标会自动出现在仪表盘中。

Q: 如何配置每日目标数？

A: 当前版本每日目标固定为 5 件，后续版本将支持自定义设置。
